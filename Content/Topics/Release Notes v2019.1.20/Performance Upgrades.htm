Performance Upgrades

Performance upgrades have been made throughout Demand Management, including the following areas:

  • Allocation - Addition of Allocation Engine Chain (ALPChain) to the workload queue as a new process type
  • API - Improved Group Import Performance
  • API - Entity Attribute Link Post temp table performance improvement
  • Assortment - Sales Plan calculations
  • Database - Performance improved by using a persisted table and index in the database
  • Events - The system will no longer rebuild table indexes prior to the output of Event information.
  • Engine - Reduced excessive engine processing times
  • Events - Optimized the refreshing of Event Slot Sets.
  • Filters - Index improvements
  • Filters - Modified udf_Common_Filter_BuildFilterConditionString.sql
  • Inventory Policy - Improved performance when using inventory dynamic policies
  • Item Manager - Improved Data Select Query Performance
  • Price Manager - Performance improvements to the Price Action Validation log
  • Price Manager - Performance improvements to Price Action calculations
  • Price Manager - Performance improvements to Price Action creation
  • Profiles - Improved Cache Refresh and Apply-step Performance
  • Promotions - Optimized the cloning of Promotions
  • Promotions - Optimized the creation of Promotions
  • Site Manager - Site Exclusion Index optimization
  • Supplier Manager - High-value Index Enhancement on Supplier Order lines
  • Workload Queue - Queued Item performance has been improved

For the complete set of performance improvements, see the Resolved Issues Log.
