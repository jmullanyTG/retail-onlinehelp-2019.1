Breaking Changes

Assortment: Excluded sites remain excluded when the cluster is changed DMDEV-59371

Sites that are specifically excluded within Assortment planning now remain so. This is based on a new system policy. If the new system policy is enabled, then when a site that has been excluded moves from one cluster to another cluster it remains excluded.

Policy Location
Allow Strict Site Exclusion Maintenance > Policies > Module: Assortment

Actualization maintenance DMDEV-51459

A new maintenance step (method assortment_Maintenance_SystemPolicy_CurrencyIDs) must be added in weekly processing before actualization.

Workflow Manager DMDEV-57224 Recommendation to create new User ID to run the nightly Workflow Heartbeat job

The trigger method "Auto Advance Node Status by Completion Days Out for Workflow Triggers" used the User ID that last modified the promotion or price when determining whether to automatically advance a status to the next status. The userâ€™s permissions and ability to modify promotions were modified as a result of the status being advanced.

The system now uses the specific User ID that is attached to the Workflow Heartbeat. By default, this User ID is set to 1 (IOAdmin). To avoid unintentionally changing permissions on IOAdmin, it is recommended that a separate User ID be created and attached to the Workflow Heartbeat.
