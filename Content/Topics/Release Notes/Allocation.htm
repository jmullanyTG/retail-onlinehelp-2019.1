Allocation

The following enhancements are now available in the Allocation solution.

Creating Save Points for Allocation Strategies DMDEV-55963

A new column on the Allocation Strategy screen labeled Create Save Point gives users the ability to create a Save Point. If a user selects Create Save Point for a particular strategy and then clicks Save, the system takes all the information relating to the strategy including location exceptions and saves the setings for the strategy. Users can then go to the Allocation Strategy Save Point screen and mass apply a Save Point to all the variants in the Item Filter.

The Create Save Point Before Restore function creates another Save Point saving variant settings before a restore.

Demand Transference

Users can now consider Demand Transference within the Allocations solution. To include or exclude this logic, navigate to Management > Inventory Manager and select the Consider Demand Transference checkbox for one or more items.

Product Model Calculation

By default (system policy Allocate.ProductModel.UseAdvanced is set to False), the product model calculation uses the MEDIAN to produce the average values. If the new system policy Allocate.ProductModel.UseAdvanced is set to True, then the calculation will use the MEAN.

Two new columns have been added to the Product Model grid

The new column Include Inner Zeros lets you selectively include or exclude inner zero periods from the consideration sample. New column Include Lost Sales allows for the inclusion of lost sales in the product model calculation, potentially preventing a situation wherein a store was out of stock and thus negatively affecting the product model.
