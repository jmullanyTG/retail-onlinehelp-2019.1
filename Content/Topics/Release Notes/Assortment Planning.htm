Assortment Planning

Clusters with zero sites DMDEV-56801

Two new columns (Health Error and Health Error Details) and a health indicator (Health %) have been added to provide more visibility if a cluster has zero sites in a time frame so assortments can be better planned.

Assorting based on future cluster sets DMDEV-56800

Functionality has been added to allow assorting based on current and future cluster sets, depending on the time period being worked with, to avoid assortment plans being over- or under-estimated.
