Workflow Manager

The following enhancement is now available in the Workflow Manager solution.

User Blacklist Rights and the nightly Workflow Heartbeat job DMDEV-57224

The system now takes into account the User Blacklist Rights that is used for the nightly Workflow Heartbeat job to determine if a Price Action, Promotion, or Event should be automatically advanced to the next Workflow Status based on the appropriate solution object's current Status State:

The system will evaluate the user's User Right to 'Modify Status' based on the solution object's current Status’ State:

  • If the Job User does not have User Rights to 'Modify Status' based on the current Status State nor permission to the Transition, the system will not change to the next Workflow Status.

  • If the Job User has User Rights to 'Modify Status' based on the current Status State AND permission to the Transition, the system will change to the next Workflow Status.
